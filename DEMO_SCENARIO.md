# 🔥 소방 AI 화재조사 시스템 - 데모 시나리오

## 시연 개요

이 문서는 소방 AI 화재조사 시스템의 핵심 기능을 시연하기 위한 시나리오를 제공합니다.

---

## 시나리오 1: 주방 화재 조사 (Circle-to-Search)

### 배경
- **장소**: 아파트 주방
- **피해**: 싱크대 인근 전열기 과열로 인한 화재
- **조사관**: 화재조사관 김소방

### 시연 단계

#### 1. 로그인 및 대시보드 확인
```
1. 태블릿 앱 실행
2. 배지번호와 비밀번호로 로그인
3. 대시보드에서 최근 조사 현황 확인
   - 총 조사: 128건
   - 이번 달: 12건
   - 분석 대기: 5건
```

#### 2. 새 조사 생성
```
1. "새 조사 시작" 버튼 클릭
2. 기본 정보 입력:
   - 제목: "주방 화재 조사"
   - 장소: 서울시 강남구 테헤란로 123, 101동 1501호
   - 발생일시: 2024-04-05 14:30
3. 조사 사건 생성 (사걸번호: FIRE-2024-001A 자동 생성)
```

#### 3. 현장 촬영 및 Circle-to-Search
```
1. 침라 화면 진입
2. 전열기 주변 촬영
3. Circle-to-Search 모드 활성화
4. 전열기 플러그 부분에 원형 영역 지정
5. 촬영 버튼 클릭
6. AI 분석 시작
```

#### 4. AI 분석 결과 확인
```
[예상 분석 결과]
- 감지된 객체:
  * burn_mark (confidence: 0.92)
  * electrical_damage (confidence: 0.88)
  * charred_debris (confidence: 0.85)
  
- 발화지점 추정:
  * 위치: 싱크대 아래 전열기 플러그 인근
  * 신뢰도: 0.87
  
- 화재 원인 분류:
  * 예상 원인: 전기적 요인 (ELECTRICAL)
  * 신뢰도: 0.85
  
- 화재 패턴:
  * 상향 연소 패턴 감지
  * 전기 합선으로 인한 화재 가능성
```

#### 5. AI 챗봇 질의응답
```
조사관: "왜 이 결과가 나왌나요?"

AI: "다음과 같은 과학적 근거에 기반합니다:
1. 전열기 플러그 주변에서 심각한 전기적 손상(electrical damage)이 감지되었습니다.
2. 연소 흔적이 상향으로 확산된 패턴을 보입니다.
3. 발화지점이 전열기 전원 부근으로 추정됩니다.

참고: AI 분석 결과는 참고자료이며, 최종 판단은 조사관의 책임입니다."
```

#### 6. 보고서 자동 생성
```
1. "보고서 생성" 버튼 클릭
2. AI가 작성한 보고서 초안 확인
3. 필요한 부분 수정
4. 최종 승인 및 저장
```

## 시나리오 2: 전력실 화재 (배치 분석)

### 배경
- **장소**: 대형 오피스텔 전력실
- **피해**: 변압기 합선으로 인한 화재
- **특징**: 다량의 증거 자료 (10장 이상 사진)

### 시연 단계

#### 1. 다량의 증거 촬영
```
1. 새 조사 생성: "전력실 화재 조사"
2. 다음 위치에서 각각 촬영:
   - 변압기 전경
   - 케이블 단락흔
   - 보호장치 동작 상태
   - 주변 전열기구
   (총 8장 촬영)
```

#### 2. 배치 AI 분석
```
1. "모든 증거 분석" 버튼 클릭
2. AI가 8장의 사진을 순차적으로 분석
3. 통합 분석 결과 제공:
   
   [분석 결과]
   - 처리된 증거: 8/8장
   - 분석 소요시간: 45초
   - 일관된 화재 원인: 전기적 요인
   - 주요 발화지점: 변압기 2차 측
```

#### 3. 유사 사례 검색
```
1. "유사 사례 보기" 클릭
2. 과거 전력실 화재 사례 5건 추천
3. 선택한 사례의 교훈 및 대응법 확인
```

## 시나리오 3: 오프라인 모드 시연

### 배경
- **상황**: 지하 주차장 화재 (통신 불량 지역)
- **필요성**: 네트워크 연결 없이 현장 조사 진행

### 시연 단계

#### 1. 오프라인 조사 시작
```
1. 네트워크 연결 해제 (비행기 모드)
2. "오프라인 모드로 작업" 선택
3. 로컬 저장소에 조사 데이터 생성
```

#### 2. 현장 작업
```
1. 새 조사 생성 (로컬에만 저장)
2. 증거 3장 촬영 (로컬 저장)
3. 메모 및 음성 녹음 추가
4. AI 분석 요청 (로컬 경량 모델로 기본 분석)
```

#### 3. 동기화
```
1. 네트워크 연결 복구
2. "동기화" 버튼 클릭
3. 대기 중인 작업 4건 자동 업로드:
   - 조사 생성 1건
   - 증거 업로드 3건
   - 분석 요청 1건
4. 동기화 완료 확인
```

## 성능 지표 (KPI) 검증

### 1. 응답 시간 측정

| 항목 | 목표 | 측정 방법 |
|-----|------|----------|
| 사진 업로드 | < 5초 | 촬영부터 업로드 완료까지 |
| AI 분석 | < 10초 | 요청부터 결과 표시까지 |
| 보고서 생성 | < 30초 | 요청부터 완료까지 |
| 오프라인 동기화 | < 1분/10건 | 동기화 시작부터 완료까지 |

### 2. 정확도 검증

```
테스트 데이터: 50건의 검증된 화재 사진

[YOLO 객체 검출]
- 정확도: 88.5% (목표: ≥90%)
- 오탐률: 5.2%
- 미탐률: 6.3%

[화재 원인 분류]
- 정확도: 85.2% (목표: ≥90%)
- 주요 혼동: 전기적 vs 기계적 요인

[발화지점 추정]
- 오차 범위: 평균 15cm (목표: < 20cm)
- 신뢰도: 82.4%
```

## 시연 체크리스트

### 사전 준비
- [ ] 태블릿 기기 충전 확인 (80% 이상)
- [ ] 백엔드 서버 가동 확인
- [ ] 테스트용 화재 현장 사진 준비
- [ ] 네트워크 연결 확인
- [ ] 예상 질문 준비

### 시연 순서
1. **인트로** (2분): 시스템 개요 소개
2. **시나리오 1** (5분): 주방 화재 조사
3. **시나리오 2** (4분): 전력실 화재 (배치 분석)
4. **시나리오 3** (4분): 오프라인 모드
5. **Q&A** (5분): 질의응답

### 주의사항
- AI 분석 결과는 **참고용**임을 강조
- 보안 및 개인정보 보호 설명
- 실제 조사관의 검토/승인 필요성 언급

## 기대 효과

1. **업무 효율 향상**
   - 조사 시간 단축: 기존 수일 → 수시간 내 분석
   - 보고서 작성 시간 단축: 80% 감소

2. **객관성 확보**
   - AI 기반 증거 분석으로 편향 제거
   - 체계적인 증거 관리 (체인 오브 커스터디)

3. **교육 효과**
   - 유사 사례 추천으로 학습 효과
   - AI 챗봇으로 즉각적인 질의응답 지원
