## 🎯 개요

소방 현장의 화재 조사 역량을 획기적으로 강화하기 위한 AI·디지털 지원체계입니다. 최신 이미지 인식(YOLO) 및 생성형 AI(ChatGPT) 기술을 활용하여 현장 조사관이 사진·음성·텍스트를 입력하면 AI가 화재 원인과 발화지점을 자동 분석하고, 조사보고서·설명·질문 응답을 생성하여 업무를 지원합니다.

### 핵심 가치

- **정확도 향상**: AI 기반 증거 분석으로 객관적인 판단 지원
- **업무 효율**: 수주 걸리던 조사를 수시간 내로 단축
- **디지털 전환**: 증거 아카이브 구축 및 데이터 기반 의사결정

## ✨ 기능 목록

### 1단계: 현장 영상 업로드 📸

| 기능 | 설명 |
|------|------|
| **파일 업로드** | 클릭 또는 드래그앤드롭으로 이미지 업로드 |
| **이미지 미리보기** | 업로드된 이미지 실시간 표시 |
| **반응형 UI** | 모바일/태블릿/데스크톱 지원 |

### 2단계: Circle-to-Search 🔍

| 기능 | 설명 |
|------|------|
| **영역 선택** | 마우스 드래그로 원형 영역 선택 |
| **시각화** | 선택 영역 실시간 하이라이트 |
| **좌표 추출** | 상대 좌표(0~1)로 변환 저장 |

### 3단계: 증거물 식별 분석 📋

| 기능 | 설명 | 출력 |
|------|------|------|
| **형상 분석** | 객체의 외형 특징 식별 | 원통형, 코일 형태 등 |
| **재질 분석** | 구성 물질 추정 | 금속, 세라믹, PVC 등 |
| **용도 분석** | 사용 목류 추정 | 가열용, 전력 공급 등 |
| **신뢰도** | 분석 결과 신뢰도 표시 | 0~100% |

### 4단계: AI 원인 분석 🤖

| 기능 | 설명 |
|------|------|
| **원인 추정** | 전기적/기계적/화학적 원인 분류 |
| **발화지점 추정** | 선택 영역 기반 발화 위치 추정 |
| **과학적 근거** | ChatGPT 기반 상세 설명 생성 |
| **권장 사항** | 후속 조치 방안 제시 |

### 5단계: 보고서 생성 📝

| 기능 | 설명 |
|------|------|
| **자동 작성** | 분석 결과를 바탕으로 보고서 초안 생성 |
| **표준 형식** | 소방청 공식 보고서 형식 준수 |
| **다운로드** | 텍스트 형태로 제공 (향후 PDF 지원 예정) |

## 🚀 설치 및 실행

### 로컬 개발 환경

```bash
# 1. 저장소 클론
git clone https://github.com/your-org/fire-investigation-ai.git
cd fire-investigation-ai/web-service

# 2. 가상환경 생성 및 활성화
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 3. 의존성 설치
pip install -r requirements.txt

# 4. 서버 실행
python main.py 3000

# 5. 브라우저에서 접속
open http://localhost:3000/static/index.html
```

## 🌐 배포 가이드 (Cafe24)

### 1. 파일 업로드

```bash
# Cafe24 서버에 SFTP/SSH로 접속
ssh user@cafe24-server.com

# 프로젝트 디렉토리 생성
mkdir -p ~/fire-ai/web-service
cd ~/fire-ai/web-service

# 파일 업로드 (로컬에서 실행)
scp -r ./* user@cafe24-server.com:~/fire-ai/web-service/
```

### 2. Python 가상환경 설정

```bash
# 원격 서버에서
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 3. 서비스 등록 (systemd)

```bash
# /etc/systemd/system/fire-ai.service 생성
sudo nano /etc/systemd/system/fire-ai.service
```

```ini
[Unit]
Description=Fire Investigation AI Service
After=network.target

[Service]
Type=simple
User=your-user
WorkingDirectory=/home/your-user/fire-ai/web-service
Environment=PATH=/home/your-user/fire-ai/web-service/venv/bin
ExecStart=/home/your-user/fire-ai/web-service/venv/bin/python main.py 80
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
# 서비스 시작
sudo systemctl daemon-reload
sudo systemctl enable fire-ai
sudo systemctl start fire-ai

# 상태 확인
sudo systemctl status fire-ai
sudo journalctl -u fire-ai -f
```

### 4. nginx 리버스 프록시 (선택)

```nginx
server {
    listen 80;
    server_name fire-ai.your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## 📚 API 문서

### Endpoints

| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/api/health` | 헬스체크 |
| POST | `/api/analyze` | 이미지 분석 |
| POST | `/api/ai-analysis` | AI 원인 분석 |
| POST | `/api/report` | 보고서 생성 |

### 예시: 이미지 분석

```bash
curl -X POST "http://localhost:3000/api/analyze" \
  -H "Content-Type: multipart/form-data" \
  -F "file=@fire_scene.jpg" \
  -F "center_x=0.5" \
  -F "center_y=0.5" \
  -F "radius=0.2"
```

**응답:**
```json
{
  "status": "success",
  "objects": [
    {
      "type": "전열기",
      "shape": "원통형 가열 장치",
      "material": "금속 + 세라믹",
      "purpose": "액체 가열용",
      "confidence": 0.92
    }
  ]
}
```
